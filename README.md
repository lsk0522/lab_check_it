# 카메라를 이용한 실습실 보호장비 착용 감지 시스템

## 1. 목적
CNC 밀링머신, 레이저 커팅기 등 위험 공작기계가 설치된 실습실에서 작업자가 마스크 및 보안경을 착용하였는지 카메라로 자동 감지하여 보호장비 착용을 유도하는 것을 목적으로 합니다. 딥러닝 기반 객체 인식(YOLOv5)으로 착용 여부를 실시간으로 판별하고, 미착용 시 시각·청각 경보를 출력하여 담당자 부재 시에도 안전 기준을 자동으로 유지하고자 합니다.

## 2. 제작 배경
실습실 장비 사용 시 보호장비를 착용을 안하고 사용하는 것을 방지하기 위함입니다.
실습실에서 CNC, 레이저 커팅기 사용 시 마스크, 보안경 착용은 필수 안전 수칙입니다. 그러나 현재 대부분의 교육기관에서는 담당자가 입장자를 직접 육안으로 확인하는 아날로그 방식에 의존하고 있습니다. 이로 인해 담당자 부재 시 미착용자 입장, 다수 동시 입장 시 개별 확인 누락, 착용 현황 기록 부재 등의 문제가 발생합니다.

산업 현장에서는 이미 AI 기반 PPE 감지 시스템이 도입되고 있으나, 교육기관 실습실 수준의 저비용, 소형화된 시스템은 찾아보기 어렵습니다. 본 프로젝트는 ROS1 Noetic, YOLOv5, 그리고 저가형 Picam 카메라를 활용해 이 문제를 해결하고자 합니다.

## 3. 세부 개발 계획

### 가. 하드웨어 활용 계획
- **PC (Ubuntu 20.04, ROS1 Noetic)**: 전체 시스템 제어 및 딥러닝 추론
- **USB 카메라 (웹캠 1080p)**: 실습실 입구에서 입장자 얼굴 실시간 촬영
- **LED 초록/빨간 (5mm × 2)**: 착용 완료(초록) / 미착용(빨간) 시각 표시
- **능동 부저 (Buzzer)**: 미착용 감지 시 청각 경보 출력
- **Arduino Uno**: ROS로부터 신호를 받아 LED·부저 GPIO 출력 제어
- **거치대 (아크릴 or 3D 프린팅, 직접 설계)**: 출입문 측면에 카메라와 LED를 하나의 하우징으로 고정

**하드웨어 연결 구성도:**
```text
[USB 카메라] → USB → [PC / ROS 시스템]
                            |
                       Serial 통신
                            |
                     [Arduino Uno]
                      ├── 초록 LED → PASS (마스크 + 보안경 모두 착용)
                      ├── 빨간 LED → FAIL (미착용)
                      └── 부저    → FAIL (미착용 경보음)
```

### 나. 소프트웨어 제작 계획
**개발 환경:**
- OS: Ubuntu 20.04 LTS (WSL)
- 미들웨어: ROS1 Noetic
- 언어: Python 3.11
- 딥러닝 프레임워크: YOLOv5 (Ultralytics)
- 영상 처리: OpenCV, cv_bridge

**ROS 노드 구성 및 데이터 흐름:**
```text
[usb_cam 노드]
      ↓  /picam/image_raw
[yolo_detector 노드]  ← YOLOv5 모델 추론
      ↓  /ppe_detection/result
[ppe_judge 노드]  ← PASS / PARTIAL / FAIL 판단
    ↙          ↓          ↘
[gpio_ctrl]  [gui_node]  [web_server]
(LED+부저)   (rqt 화면)  (Flask 대시보드)
```

**감지 클래스 (4종):**
- `mask` : 마스크 착용
- `no_mask` : 마스크 미착용
- `goggles` : 보안경 착용
- `no_goggles` : 보안경 미착용

**판단 로직:**
- **PASS**: 마스크 착용 + 보안경 착용 → 초록 LED 점등
- **PARTIAL**: 둘 중 하나만 착용 → 노란 LED + 안내 메시지
- **FAIL**: 마스크 미착용 + 보안경 미착용 → 빨간 LED + 부저 경보

**딥러닝 모델 학습:**
- 데이터셋: Roboflow PPE 공개 데이터셋 (44,000장 이상)
- 학습 방법: YOLOv5s 사전학습 모델 기반 전이학습 (Transfer Learning)
- 학습 환경: Google Colab (무료 T4 GPU)
- 학습 파라미터: Epoch 100, Batch 16, Image size 640x640

**GUI / Web:**
- **GUI**: `rqt_image_view`로 실시간 감지 화면 및 착용률 통계 표시
- **Web**: `rosbridge_server` + Flask로 웹 대시보드 구현 (실시간 카메라 피드 + 통과/거부 인원 현황)

### 다. 개발 일정
- **1주차: 환경 구축 및 데이터 준비**
  - ROS Noetic + usb_cam 설치 및 카메라 동작 확인
  - Roboflow에서 PPE 데이터셋 수집 및 클래스 정리
- **2주차: 모델 학습 및 핵심 노드 개발**
  - Google Colab에서 YOLOv5 전이학습 수행 및 성능 검증
  - `yolo_detector`, `ppe_judge` ROS 노드 작성
- **3주차: 입출력 및 연동 구현**
  - Arduino GPIO 연동 (LED + 부저)
  - rqt GUI 구현
  - Flask 웹 대시보드 구현
- **4주차: 통합 테스트 및 완성**
  - 전체 노드 통합 실행 및 4가지 케이스 성능 측정 (모두 착용 / 마스크만 / 보안경만 / 둘 다 미착용)
  - 거치대 제작, 발표 자료 준비

### 라. ROS 패키지 구조
```text
ppe_system/                     # 패키지 루트
├── launch/
│   └── ppe_full.launch         # 전체 시스템 실행
├── src/
│   ├── yolo_detector.py        # YOLO 추론 노드
│   ├── ppe_judge.py            # PASS/FAIL 판단 노드
│   ├── gpio_ctrl.py            # Arduino GPIO 제어 노드
│   └── web_server.py           # Flask 웹 서버 노드
├── msg/
│   └── PPEResult.msg           # 커스텀 메시지 정의
├── models/
│   └── ppe_yolov5s.pt          # 학습된 YOLO 가중치
└── web/
    └── templates/index.html    # 웹 대시보드
```

## 4. 프로젝트에 활용할 기본 이론

### 가. 하드웨어 기본 이론
- **Picam 카메라 (`usb_cam` 또는 `picam_cam`)**: ROS1의 카메라 노드를 통해 카메라 영상을 `/picam/image_raw` 토픽으로 발행합니다. `sensor_msgs/Image` 메시지 타입을 사용하며, `cv_bridge` 라이브러리를 통해 OpenCV 이미지로 변환하여 처리합니다.
- **GPIO (General Purpose Input/Output)**: Arduino Uno의 디지털 출력 핀으로 LED와 부저를 제어합니다. ROS 노드에서 PASS/FAIL 신호를 Serial 통신으로 Arduino에 전달하면, Arduino가 해당 핀에 HIGH/LOW 신호를 출력하여 LED와 부저를 동작시킵니다.
- **기구 설계 (거치대)**: 카메라가 입장자의 얼굴 영역(마스크·보안경)을 정면에서 포착할 수 있도록 출입문 측면 벽면에 고정되는 브라켓을 직접 설계합니다. 카메라와 LED를 하나의 하우징에 통합하여 설치 편의성을 높입니다.

### 나. 소프트웨어 기본 이론
- **ROS1 (Robot Operating System)**: 분산 노드(Node) 구조로 각 기능을 독립적으로 구현하며, 토픽(Topic) 기반 발행/구독(Publish/Subscribe) 통신으로 노드 간 데이터를 전달합니다. `roslaunch`를 사용하면 여러 노드를 한 번에 실행하고 파라터를 관리할 수 있습니다.
- **YOLOv5 (You Only Look Once v5)**: 이미지를 한 번만 처리하는 단일 단계 객체 탐지(One-Stage Detector) 알고리즘으로 실시간 처리에 적합합니다. 이미지를 격자(Grid)로 분할하여 각 셀에서 바운딩 박스 좌표와 클래스 확률을 동시에 예측합니다.
- **전이학습 (Transfer Learning)**: COCO 데이터셋으로 사전 학습된 YOLOv5s 가중치를 초기값으로 사용하여 PPE 데이터셋으로 추가 학습(파인튜닝)합니다. 처음부터 학습하는 것보다 훨씬 적은 데이터와 시간으로 높은 인식 성능을 달성할 수 있습니다.
- **OpenCV + cv_bridge**: `cv_bridge`는 ROS 이미지 메시지와 OpenCV 이미지 간 변환을 담당합니다. OpenCV는 감지 결과 바운딩 박스 시각화, 영상 리사이즈 등 전후처리에 활용됩니다.
- **NMS (Non-Maximum Suppression)**: 동일 객체에 대해 여러 바운딩 박스가 생성될 때, IoU(Intersection over Union) 기준으로 겹치는 박스를 제거하고 신뢰도가 가장 높은 박스만 남기는 후처리 기법입니다.

## 5. 기대효과

### 가. 프로젝트 제작으로써 기대되는 효과
본 시스템을 통해 실습실 입장 전 보호장비 착용 여부를 자동으로 감지함으로써 담당자 부재 시에도 안전 기준이 유지되어 실습실 내 안전사고를 예방할 수 있습니다. 또한 반복적인 착용 확인 업무가 자동화되어 담당자의 업무 부담이 경감되고, 착용 현황 데이터가 누적되어 안전 교육의 효과를 수치로 확인할 수 있습니다.

### 나. 카메라 활용 역량의 발전
ROS1의 `usb_cam` (또는 `picam_cam`) 패키지를 활용하여 실시간 영상 스트림을 ROS 토픽으로 처리하고, `cv_bridge`를 통해 OpenCV와 연동하는 과정을 직접 구현함으로써 카메라 기반 영상 처리 시스템 설계 역량을 키울 수 있습니다.

### 다. 딥러닝 실무 역량의 발전
YOLOv5 전이학습의 전 과정(데이터 수집 → 라벨링 → 학습 → 검증 → 배포)을 직접 수행하며 AI 모델 개발의 실무적인 흐름을 경험하고, 객체 인식 모델의 성능을 개선하는 능력을 기를 수 있습니다.

### 라. ROS 시스템 설계 역량의 발전
다수의 노드가 토픽으로 연결되는 분산 시스템을 직접 설계하고 디버깅하는 과정을 통해 ROS1 기반 로봇 소프트웨어 아키텍처 설계 역량을 향상시킬 수 있습니다.

### 마. HW·SW 통합 및 문제 해결 역량의 발전
Arduino GPIO와 ROS 노드를 Serial 통신으로 연동하는 임베디드-소프트웨어 통합 개발을 경험하고, 실제 실습실이라는 현실 문제를 정의하여 기술로 해결하는 과정을 통해 공학적 사고력과 문제 해결 역량을 기를 수 있습니다.
